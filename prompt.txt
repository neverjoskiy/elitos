  Implementation Plan - Гибридное ядро ОС (C + Assembly)

  Problem Statement:

  Создание учебного гибридного ядра ОС для x86/x86_64 с вытесняющей многозадачностью, виртуальной файловой системой и простой
  СУБД. Проект рассчитан на долгосрочное развитие с акцентом на обучение.

  Requirements:

  - Язык: C + Assembly (граница 95/5)
  - Платформа: x86/x86_64
  - Запуск: Реальное железо (с эмулятором для отладки)
  - Архитектура: Гибридное ядро с загружаемыми модулями
  - Многозадачность: Вытесняющая
  - БД: Полноценная СУБД (транзакции, индексы, SQL)
  - ФС: Виртуальная (в памяти)
  - Стадия: Постепенное улучшение базового прототипа

  Background:

  Для новичка в системном программировании критически важно начать с минимального рабочего примера и постепенно добавлять
  функционал. Ключевые этапы:

  1. Загрузчик (bootloader) — 512 байт, загружает ядро
  2. Защищённый режим x86 — инициализация памяти, таблицы страниц
  3. Прерывания — обработка системных вызовов
  4. Планировщик — вытесняющая многозадачность
  5. Виртуальная ФС — простая структура в памяти
  6. Терминал — ввод/вывод
  7. Команды — базовые утилиты
  8. СУБД — простая структура, постепенно усложняется

  Proposed Solution:

  Поэтапная реализация с акцентом на "работающий прототип на каждом шаге":

  - Шаг 1-3: Загрузчик + защищённый режим + терминал (Hello World)
  - Шаг 4-6: Планировщик + простая ФС + базовые команды
  - Шаг 7-9: СУБД (начинаем с простой хэш-таблицы, добавляем индексы/транзакции)
  - Шаг 10+: Модульная архитектура, расширяемость

  ───────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────

  Task Breakdown:

  Task 1: Создать загрузчик (bootloader) в 16-битном реальном режиме
  - Objective: Написать 512-байтный загрузчик, который загружает ядро по адресу 0x1000
  - Implementation: Assembly (NASM), загрузка второго загрузчика или напрямую ядра
  - Test: Запуск в QEMU, вывод "Loading kernel..." на экран
  - Demo: Эмулятор показывает сообщение о загрузке

  Task 2: Переключение в защищённый режим 32/64-bit
  - Objective: Инициализировать GDT, включить PAE, переключиться в защищённый режим
  - Implementation: Assembly для переключения, C для инициализации
  - Test: Проверка CR0 регистра, вывод "Protected mode enabled"
  - Demo: QEMU показывает переход в защищённый режим

  Task 3: Инициализация памяти (paging)
  - Objective: Настроить таблицы страниц, включить пейджинг
  - Implementation: C для создания таблиц, assembly для переключения
  - Test: Проверка доступа к памяти, вывод "Memory initialized"
  - Demo: Простая программа печатает сообщение из высокого адреса

  Task 4: Обработка прерываний (IDT)
  - Objective: Настроить IDT, обработать деление на ноль, системные вызовы
  - Implementation: Assembly для обработчиков, C для логики
  - Test: Генерация исключений, проверка обработки
  - Demo: Программа вызывает деление на ноль, ядро обрабатывает

  Task 5: Простой терминал (conio)
  - Objective: Вывод текста на экран, базовый ввод
  - Implementation: C, прямой доступ к видеопамяти (0xB8000)
  - Test: Печать "Hello, Kernel!" и чтение символа с клавиатуры
  - Demo: Терминал печатает приветствие и реагирует на ввод

  Task 6: Планировщик задач (базовый)
  - Objective: Реализовать простой round-robin планировщик с 2 задачами
  - Implementation: C, сохранение/восстановление контекста (registers)
  - Test: Две задачи поочерёдно печатают сообщения
  - Demo: Два "потока" выводят текст параллельно

  Task 7: Виртуальная файловая система (VFS)
  - Objective: Простая структура в памяти — дерево каталогов, файлы
  - Implementation: C, структуры данных (списки/деревья)
  - Test: Создание файла, чтение, запись, удаление
  - Demo: Программа создаёт файл "test.txt", читает и печатает содержимое

  Task 8: Базовые команды (ls, cd, cat, echo)
  - Objective: Реализовать простые команды терминала
  - Implementation: C, парсинг командной строки
  - Test: Ввод команд, проверка вывода
  - Demo: Пользователь вводит ls, создает файл через echo, читает через cat

  Task 9: Простая СУБД (хэш-таблица)
  - Objective: Базовая СУБД с хэш-таблицей для хранения данных
  - Implementation: C, структуры данных, простой SQL-парсер (SELECT/INSERT)
  - Test: Создание таблицы, вставка данных, выборка
  - Demo: Программа создает таблицу users, добавляет запись, выводит её

  Task 10: Улучшенная СУБД (индексы)
  - Objective: Добавить индексы для ускорения поиска
  - Implementation: C, структуры данных (деревья/хэш-индексы)
  - Test: Сравнение скорости с индексом и без
  - Demo: Программа показывает ускорение поиска по индексу

  Task 11: Улучшенная СУБД (транзакции)
  - Objective: Реализовать транзакции (BEGIN/COMMIT/ROLLBACK)
  - Implementation: C, журнал изменений (undo/redo)
  - Test: Откат изменений при ошибке
  - Demo: Программа делает несколько изменений, откатывает их

  Task 12: Модульная архитектура
  - Objective: Разделить ядро на модули (ядро, ФС, СУБД, терминал)
  - Implementation: C, загрузка модулей, интерфейсы
  - Test: Загрузка модулей во время выполнения
  - Demo: Программа загружает модуль "network" или "filesystem"

  Task 13: Интеграция и тестирование
  - Objective: Объединить все компоненты, протестировать
  - Implementation: C, тесты для каждого модуля
  - Test: Полный цикл: загрузка → терминал → команда → БД → вывод
  - Demo: Рабочая ОС с терминалом, командами и базой данных
